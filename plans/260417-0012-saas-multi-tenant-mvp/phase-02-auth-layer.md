# Phase 02: Auth Layer — Better Auth + Polar Plugin

## Context Links
- Parent: [plan.md](./plan.md)

- Depends on: Phase 01 (D1 + Hono)

## Overview
Install Better Auth with D1 backend, email+password + Google OAuth. Add Polar plugin for billing sync (webhook routing + customer auto-creation). Auth middleware protects `/api/*` routes. Login/signup UI in Preact SPA.

## Key Insights
- Better Auth must be initialized INSIDE `fetch()` handler — no `process.env` in CF Workers
- D1 adapter: `{ db: env.DB, type: "sqlite" }` — native, no Drizzle needed for MVP
- Polar plugin auto-routes webhooks to `/api/auth/polar/webhooks`
- `createCustomerOnSignUp: true` → Polar customer created on registration
- Cookie-based sessions stored in D1 `session` table — simplest for SaaS dashboard
- Schema generated by `npx better-auth generate` — use its SQL as migration source of truth

## Requirements

### Functional
- User can sign up with email+password
- User can sign in with Google OAuth
- Authenticated sessions via cookies (D1-backed)
- Protected API routes return 401 for unauthenticated requests
- Polar customer auto-created on signup
- Polar webhook endpoint receives and validates billing events
- Login/signup pages in Preact SPA

### Non-Functional
- Session lookup < 50ms (D1 is edge-local)
- Auth initialization < 10ms per request (lazy singleton pattern)
- No leaked credentials in client-side code

## Architecture

```
Preact SPA                    CF Worker (Hono)
  Login page ──POST──>  /api/auth/sign-in/email
  Signup page ─POST──>  /api/auth/sign-up/email
  Google btn ──GET───>  /api/auth/sign-in/social (google)
                             │
                        Better Auth
                             │
                        D1 (user, session, account tables)
                             │
                        Polar plugin
                             │
                        Polar API (createCustomer on signup)

Polar Dashboard ──webhook──> /api/auth/polar/webhooks
                             │
                        onCustomerStateChanged → update polar_subscription
                        onOrderPaid → grant tier access
```

## Implementation Steps

### 1. Install dependencies
```bash
npm install better-auth @polar-sh/better-auth @polar-sh/sdk
```

### 2. Generate Better Auth schema
```bash
npx better-auth generate
```
Compare output with `0001_initial_schema.sql`. If differences, create `0002_better_auth_schema.sql` with ALTER statements or replace 0001 if not yet applied to production.

### 3. Create `src/api/auth.js` (<120 lines)
```js
// Lazy singleton pattern — init once per isolate, re-use across requests
let _auth = null;

export function getAuth(env) {
  if (_auth) return _auth;
  _auth = betterAuth({
    database: { db: env.DB, type: "sqlite" },
    baseURL: env.APP_URL,  // e.g. https://api.newsengine.app
    emailAndPassword: { enabled: true },
    socialProviders: {
      google: {
        clientId: env.GOOGLE_CLIENT_ID,
        clientSecret: env.GOOGLE_CLIENT_SECRET,
      }
    },
    plugins: [
      polar({
        client: new Polar({ accessToken: env.POLAR_ACCESS_TOKEN }),
        createCustomerOnSignUp: true,
        enableCustomerPortal: true,
        checkout: { enabled: true },
        webhooks: {
          secret: env.POLAR_WEBHOOK_SECRET,
          onCustomerStateChanged: async ({ event }) => { /* Phase 05 */ },
          onOrderPaid: async ({ event }) => { /* Phase 05 */ },
        }
      })
    ]
  });
  return _auth;
}
```

**Gotcha**: Lazy singleton works because CF Worker isolates are recycled. The `_auth` lives for the isolate lifetime. If `env` changes between requests (it won't in production), this breaks — acceptable tradeoff for MVP.

### 4. Create `src/api/middleware/auth.js` (<50 lines)
```js
export function requireAuth() {
  return async (c, next) => {
    const auth = getAuth(c.env);
    const session = await auth.api.getSession({ headers: c.req.raw.headers });
    if (!session?.user) return c.json({ error: 'Unauthorized' }, 401);
    c.set('user', session.user);
    c.set('session', session.session);
    await next();
  };
}
```

### 5. Mount auth in `src/api/app.js`
- `app.all('/api/auth/*', (c) => getAuth(c.env).handler(c.req.raw))`
- Apply `requireAuth()` middleware to `/api/streams/*`, `/api/runs/*`, `/api/billing/*`
- Legacy routes (`/trigger`, `/preview`) keep existing `TRIGGER_SECRET` auth

## Security Considerations
- Passwords hashed by Better Auth (bcrypt/scrypt internally)
- Session tokens httpOnly, secure, sameSite=lax cookies
- CSRF protection via Better Auth's built-in mechanism
- Google OAuth secrets stored via `wrangler secret put`, never in code or wrangler.toml vars
- Polar webhook signature validated by plugin automatically
- Rate limit login attempts (Hono rate-limit middleware in Phase 07)
